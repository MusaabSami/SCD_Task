Project OMG: Git Workflow Challenge

Company Context

You are a developer at Emumba, a fast-growing software company working on Project OMG, a revolutionary application that will change the way users interact with cloud-based systems.

Git Workflow Tasks

  1. Bootstrapping the Project

    Initialize a Git repository, stage all files, commit them, and push the initial version to a remote repository.

  2. First Major Edit

    Update the product documentation (doc.txt), commit the changes, and push them to the master branch.

  3. A New Feature in the Works

    Create a new branch (b1), add a new file (file2.txt), commit it, and push this branch to the remote repository.

  4. The Merge Challenge

    Switch back to master, merge b1 seamlessly, and push the final version.

  5. Keeping the Codebase Clean

    Delete branch b1 locally and remotely to prevent clutter.

  6. Marking a Milestone

    Tag the latest version as v1.0 for release. Then, as per the team lead's request, remove the tag both locally and remotely.

Note: A surprise code review is imminent. Ensure your Git history is clean, well-structured, and properly documented!
